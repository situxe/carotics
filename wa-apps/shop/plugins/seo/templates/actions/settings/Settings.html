Полезная информация

Правила, заданные в настройках плагина, работают в том случае, если мета-теги и/или описание товара, категории не заполнено, если есть необходимость перебить такие описания используйте флажок "Игнорировать персональные заголовки" в настройках плагина. (Возможность перебивать написанные правила имеется только для категорий или товаров) Если желаете отключить правило, оставьте поле пустым или снимите галочку у нужного раздела, при этом данные будут сохранены.

1) Использование модификаторов.

Список доступных модификаторов указан ниже списка доступных переменных для определенного типа посадочной страницы.

Примеры использования:

|if_page_not_first - преобразует строку в пустую если страница первая, пример {'Страница {page_number}'|if_page_not_first}

|currency - преобразует число в денежный формат, пример {product_price|currency}

{categories|sep_hyphen} - выводит весь путь категорий через тире.

Так же есть возможность использования нескольких модификаторов одновременно.

Дополнительный список:

|reverse - в обратном порядке

|sep_backslash - разделитель обратный слеш

|sep_comma - запятая

|sep_hyphen - тире

|sep_slash - слэш (по умолчанию)

|sep_space – пробел

2) Использование синонимов

Вы можете настроить формирование описаний с помощью синонимов, пример конструкции "Можно использовать синонимы, например: {'продукт','товар','изделие'}, они будут показываться рандомно, обновите страницу и увидите новое значение.

3) Вывод дополнительного описания для категорий

Для вывода на сайт дополнительного описания категории, включите возможность в настройках плагина и вставьте код {shopSeoViewHelper::getCategoryAdditionalDescription()} в шаблон категорий category.html

4) Вывод описания для тегов

Для вывода описания для тегов, сформируйте шаблон формирования описаний и вставьте код {shopSeoViewHelper::getTagDescription()} в шаблон search.html

5) Использование переменных витрин в шаблонах

Если Вы хотите использовать переменные витрин в шаблонах темы дизайна, используйте код вида {shopSeoViewHelper::parseTemplate('{storefront_name}')} где storefront_name – название переменной.

6) Дополнительные опции

1) Для более гибкой настройки оптимизации страниц каталога или карточек товара, Вы можете включить возможность указывать правила для подкатегорий или товаров определенной категории.

2) Для вывода альтернативных категорий в качестве заголовка H1, включите соответствующие галки в дополнительных опциях плагина.
